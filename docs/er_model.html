ER-модель (Сутнісно-зв’язна модель)

ER-модель описує структуру даних, що використовуються в системі управління проєктами. Вона містить основні сутності (таблиці), атрибути та зв’язки між ними. Ця модель слугує основою для побудови реляційної бази даних.

Сутності та атрибути:

  • User (Користувач)
    • user_id (PK)
    • name
    • email
    • role (admin, manager, developer)
  • Project (Проєкт)
    • project_id (PK)
    • title
    • description
    • created_at
    • deadline
    • status
    • owner_id (FK → User.user_id)
  • Task (Задача)
    • task_id (PK)
    • title
    • description
    • status
    • priority
    • deadline
    • project_id (FK → Project.project_id)
    • assignee_id (FK → User.user_id)
  • Comment (Коментар)
    • comment_id (PK)
    • content
    • created_at
    • task_id (FK → Task.task_id)
    • author_id (FK → User.user_id)
  • Attachment (Файл)
    • file_id (PK)
    • file_name
    • file_url
    • task_id (FK → Task.task_id)

Основні зв’язки:

  • Один користувач може створювати багато проєктів (1 → N).
  • Один проєкт має багато задач (1 → N).
  • Одна задача має одного виконавця (N → 1).
  • Одна задача має багато коментарів (1 → N).
  • Один користувач може залишати багато коментарів (1 → N).
  • Одна задача може мати багато вкладень (1 → N).

ER-модель дає чітке уявлення про структуру даних у системі та є основою для проєктування таблиць у СУБД (наприклад, PostgreSQL або MySQL).

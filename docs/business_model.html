Модель бізнес-об’єктів

Модель бізнес-об’єктів описує основні сутності (об'єкти) системи управління проєктами, їх атрибути та взаємозв’язки. Ця модель є основою для побудови логіки бази даних та реалізації API.

Основні бізнес-об’єкти:

  • Користувач (User) – особа, яка має доступ до системи.
    • Атрибути: ID, ім’я, email, роль (менеджер, виконавець, адміністратор).
  • Проєкт (Project) – основна одиниця планування.
    • Атрибути: ID, назва, опис, дата створення, дедлайн, статус.
  • Задача (Task) – конкретна одиниця роботи в межах проєкту.
    • Атрибути: ID, назва задачі, опис, проєкт, виконавець, пріоритет, дедлайн, статус.
  • Коментар (Comment) – повідомлення, пов’язане із задачею.
    • Атрибути: ID, автор, текст, дата створення, ID задачі.
  • Файл (Attachment) – прикріплений документ або зображення.
    • Атрибути: ID, ім’я файлу, тип, посилання, ID задачі або проєкту.

Зв’язки між об’єктами:

  • Користувач може створювати проєкти та задачі.
  • Кожен проєкт містить багато задач.
  • Задача призначається одному користувачу.
  • До задач можуть додаватися коментарі та файли.

Ця модель дозволяє легко масштабувати систему, додавати нові ролі чи функціональність, зберігаючи зрозумілу логіку структури даних.

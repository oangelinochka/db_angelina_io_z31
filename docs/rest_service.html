Що таке REST?

REST (Representational State Transfer) — це архітектурний стиль для створення веб-сервісів, які працюють через протокол HTTP. RESTful API дозволяє клієнтам (наприклад, веб-додаткам) взаємодіяти з сервером через стандартні HTTP-запити.

Основні HTTP-методи

  • GET — отримання даних (наприклад, список проєктів)
  • POST — створення нового ресурсу (наприклад, нова задача)
  • PUT — оновлення ресурсу
  • DELETE — видалення ресурсу

Приклад REST-запитів для системи управління проєктами

1. Отримати всі задачі


GET /api/tasks

JSON-відповідь:


[
  {
    "id": 1,
    "title": "Підготовка ТЗ",
    "status": "У процесі",
    "assigned_to": "Іван Петренко"
  },
  {
    "id": 2,
    "title": "Розробка бази даних",
    "status": "Не розпочато",
    "assigned_to": "Олена Коваленко"
  }
]

2. Створити нову задачу


POST /api/tasks
Content-Type: application/json

{
  "title": "Тестування API",
  "status": "У процесі",
  "assigned_to": 3,
  "project_id": 1,
  "due_date": "2025-06-20"
}

3. Оновити статус задачі


PUT /api/tasks/1
Content-Type: application/json

{
  "status": "Завершено"
}

4. Видалити задачу


DELETE /api/tasks/2

Висновок

RESTful сервіс у проєкті дозволяє інтегрувати клієнтські додатки з базою даних через HTTP-запити. Це забезпечує масштабованість, простоту та сумісність із сучасними вебтехнологіями.
